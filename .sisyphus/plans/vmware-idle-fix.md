# VMware 虚拟机摸鱼检测修复计划

## 问题
用户鼠标焦点在 VMware 虚拟机里时，被错误检测为无操作（摸鱼）。

## 原因
- `powerMonitor.getSystemIdleTime()` 只检测宿主机上的键盘/鼠标输入
- VMware 虚拟机捕获了鼠标/键盘事件，这些事件不会传递到宿主机
- 导致系统错误地将虚拟机中工作的用户判断为"空闲"

## 修复方案
完全移除系统空闲时间检测，只依赖锁屏事件。

### 原因
- 当用户在 VM 里工作时，窗口会失去焦点（因为 VM 窗口有焦点）
- 任何基于"窗口焦点"的检测都会在用户使用 VM 时错误触发
- 锁屏事件是最可靠的"用户离开"信号
- 用户在 VM 里工作不会锁屏，所以不会被误判

### 新逻辑
1. 移除 `checkIdle()` 定时检测
2. 只依赖 `powerMonitor.on('lock-screen')` 事件
3. 当系统锁屏时触发暂停
